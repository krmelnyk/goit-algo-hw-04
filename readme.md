# Домашнє завдання 4

## Завдання 3. Порівняння алгоритмів сортування

Було порівняно три алгоритми сортування:

- сортування вставками;
- сортування злиттям;
- Timsort, який використовується у вбудованій функції Python `sorted()`.

Для вимірювання часу виконання використано модуль `timeit`.
Алгоритми тестувалися на масивах розміром `1000`, `5000` і `10000`
елементів.

Для кожного розміру було використано кілька типів вхідних даних:

- випадковий масив;
- вже відсортований масив;
- масив у зворотному порядку;
- майже відсортований масив.

## Аналіз складності

Сортування вставками має середню та найгіршу складність `O(n^2)`.
Це підтверджується результатами на випадкових і зворотно відсортованих
масивах. Наприклад, для `10000` елементів випадковий масив сортувався
приблизно `1.804241` секунди, а зворотно відсортований - приблизно
`3.567651` секунди.

Сортування злиттям має складність `O(n log n)` незалежно від початкового
порядку елементів. У тестах воно працювало значно стабільніше за
сортування вставками. Для `10000` елементів час виконання залишався в межах
приблизно `0.010653`-`0.017013` секунди.

Timsort також має найгіршу складність `O(n log n)`, але на практиці працює
швидше, тому що поєднує ідеї сортування злиттям та сортування вставками.
Алгоритм знаходить вже впорядковані частини масиву і ефективно використовує
їх під час сортування. Саме тому на відсортованих і майже відсортованих
даних Timsort показав найкращі результати.

## Висновки

Емпіричні результати підтвердили теоретичні оцінки складності алгоритмів.
Сортування вставками може бути ефективним тільки для малих або майже
відсортованих наборів даних, але на великих випадкових чи зворотно
відсортованих масивах воно швидко стає повільним через складність `O(n^2)`.

Сортування злиттям значно краще масштабується і показує стабільний результат
для різних типів вхідних даних завдяки складності `O(n log n)`.

Timsort виявився найефективнішим у всіх протестованих випадках. Його
перевага особливо помітна на вже відсортованих і майже відсортованих даних.
Це пояснює, чому в більшості практичних задач у Python доцільніше
використовувати вбудовані `sorted()` або `list.sort()`, а не реалізовувати
алгоритми сортування вручну.
